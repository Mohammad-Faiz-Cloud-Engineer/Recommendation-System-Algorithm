# Code of Conduct

## Why This Matters

I built this recommendation system because I wanted to solve real problems in content ranking and personalization. Along the way, I learned that the best solutions come from diverse perspectives and honest collaboration. This Code of Conduct exists to protect that environment.

## The Standard

I expect everyone who participates in this project whether you're filing a bug report, submitting code, or just asking questions to treat others with basic human decency. That means:

- Assume good faith. Most misunderstandings aren't malicious.
- Critique ideas and code, not people.
- If you disagree, explain why. "This approach has issues because..." is useful. "This is stupid" isn't.
- Remember there's a person on the other end of your comment.

## What's Not Acceptable

I won't tolerate harassment, personal attacks, or discrimination of any kind. This includes but isn't limited to:

- Offensive comments related to gender, sexual orientation, disability, appearance, race, or religion
- Deliberate intimidation or stalking
- Unwelcome sexual attention
- Publishing others' private information without permission
- Sustained disruption of discussions

If you wouldn't say it to someone's face in a professional setting, don't say it here.

## Scope

This applies everywhere this project exists: GitHub issues, pull requests, code comments, and any other communication channels related to this work.

## Enforcement

If someone violates this Code of Conduct, I'll take whatever action seems appropriate, from a warning to a permanent ban. I'll try to be fair, but I'm also not interested in endless debates about edge cases. Use common sense.

If you experience or witness unacceptable behavior, contact me directly at my GitHub profile: [@Mohammad-Faiz-Cloud-Engineer](https://github.com/Mohammad-Faiz-Cloud-Engineer). I'll handle it privately and as quickly as I can.

## On Contributions

A few practical notes since this comes up:

- If you're new to the codebase, ask questions. I'd rather answer the same question twice than have you waste hours being confused.
- If you submit a PR that needs work, I'll tell you what needs to change and why. That's not personal—it's how we maintain quality.
- If your PR gets rejected, it's not a rejection of you. Sometimes ideas don't fit, timing is wrong, or there's context you don't have.

## The Real Goal

I want this project to be a place where people can learn, contribute, and build something useful without dealing with unnecessary drama or toxicity. That's it. Everything else is details.

If you can work within that framework, you're welcome here.
